# Documentation Naming Convention Standard

> **Universal Format**: `EXECUTIONS-{titulo}-{YYYY-MM-DD-HHmm}.md`
> **Applies To**: All generated documentation (execution reports, config files, MCP tool outputs)
> **Effective**: November 7, 2025

---

## üéØ Core Principle

**One naming pattern for all generated documentation** to ensure consistency, clarity, and automated processing.

---

## üìã Naming Convention

### Format

```
EXECUTIONS-{titulo}-{YYYY-MM-DD-HHmm}.md
```

### Components

| Component | Description | Example |
|-----------|-------------|---------|
| `EXECUTIONS` | Fixed prefix (uppercase) | `EXECUTIONS` |
| `{titulo}` | Descriptive title (kebab-case) | `audit-complete` |
| `{YYYY-MM-DD}` | ISO date | `2025-11-07` |
| `{HHmm}` | Time (24h format) | `1430` (2:30 PM) |
| `.md` | Markdown extension | `.md` |

### Examples

```
‚úÖ EXECUTIONS-audit-complete-2025-11-07-1430.md
‚úÖ EXECUTIONS-phase3c-deployment-2025-11-04-0900.md
‚úÖ EXECUTIONS-jules-integration-2025-11-03-1615.md
‚úÖ EXECUTIONS-weekly-summary-2025-11-07-2300.md

‚ùå audit-complete-cde-mcp-2025-11-07.md (old format)
‚ùå EXECUTIVE_SUMMARY_AUDIT_2025-11-07.md (old format)
‚ùå execution-phase3c-deployment-2025-11-04.md (old format)
```

## üóÇÔ∏è Application Scope

### 1. Execution Reports

**Location**: `agent-docs/execution/`

**Before** (Old Format):
```
audit-complete-cde-mcp-2025-11-07.md
EXECUTIVE_SUMMARY_AUDIT_2025-11-07.md
decision-matrix-implementation-2025-11-07.md
```

**After** (New Format):
```
EXECUTIONS-audit-complete-2025-11-07-1430.md
EXECUTIONS-audit-summary-2025-11-07-1435.md
EXECUTIONS-decision-matrix-2025-11-07-1440.md
```

---

### 2. Agent Configuration Files

**Location**: Repository root

**Current Files**: `AGENTS.md`, `GEMINI.md`

**New Standard**: Keep current names for **primary config files**, but generated variants use new format:

```
# Primary files (permanent)
AGENTS.md               ‚Üê Main agent instructions (OpenAI standard)
GEMINI.md               ‚Üê Main Gemini instructions (Google standard)

# Generated variants (timestamped)
EXECUTIONS-agents-config-update-2025-11-07-1500.md
EXECUTIONS-gemini-config-update-2025-11-07-1505.md
```

---

### 3. MCP Tool Outputs

All MCP tools generating markdown files MUST use this format:

**Example: `cde_startFeature` output**:
```python
# OLD
output_file = "agent-docs/execution/feature-implementation-2025-11-07.md"

# NEW
from datetime import datetime
timestamp = datetime.now().strftime("%Y-%m-%d-%H%M")
output_file = f"agent-docs/execution/EXECUTIONS-feature-implementation-{timestamp}.md"
```

---

### 4. Weekly Summaries

**Generated by**: GitHub Actions (weekly-cleanup.yml)

**Format**: Special case using ISO week number

```
WEEK-{YYYY-WW}.md

Examples:
WEEK-2025-45.md   (Week 45 of 2025)
WEEK-2025-46.md   (Week 46 of 2025)
```

**Why different**: Weekly summaries are consolidations, not individual executions. The `WEEK-` prefix distinguishes them from regular execution reports.

## üîß Implementation

### Step 1: Update Pre-Commit Hook

Add validation for new naming convention:

```python
# scripts/validation/validate-execution-names.py

import re
from pathlib import Path

EXECUTION_PATTERN = r"^EXECUTIONS-[\w-]+-\d{4}-\d{2}-\d{2}-\d{4}\.md$"
WEEK_PATTERN = r"^WEEK-\d{4}-\d{2}\.md$"

def validate_execution_filename(filename: str) -> bool:
    """Validate execution report filename."""
    if re.match(EXECUTION_PATTERN, filename):
        return True
    if re.match(WEEK_PATTERN, filename):
        return True
    return False

# Pre-commit hook usage
for file in Path("agent-docs/execution").glob("*.md"):
    if file.name in ["README.md", ".archive"]:
        continue

    if not validate_execution_filename(file.name):
        print(f"‚ùå Invalid filename: {file.name}")
        print(f"   Expected: EXECUTIONS-{{titulo}}-{{YYYY-MM-DD-HHmm}}.md")
        exit(1)
```

**File**: `src/mcp_tools/orchestration.py`

from datetime import datetime

def generate_execution_filename(title: str) -> str:
    """
    Generate standardized execution report filename.

    Args:
        title: Descriptive title (will be converted to kebab-case)

    Returns:
        Filename in format: EXECUTIONS-{title}-{YYYY-MM-DD-HHmm}.md

    # Convert to kebab-case

    timestamp = datetime.now().strftime("%Y-%m-%d-%H%M")

### Step 3: Migration Script

Rename existing files to new format:

```python
# scripts/migration/rename-execution-files.py

from pathlib import Path
from datetime import datetime
import re

def extract_date_from_old_format(filename: str) -> str:
    """Extract date from old format filename."""
    match = re.search(r"(\d{4}-\d{2}-\d{2})", filename)
    if match:
        return match.group(1)
    return datetime.now().strftime("%Y-%m-%d")

def generate_new_filename(old_filename: str) -> str:
    """Generate new filename from old format."""
    # Extract base name
    base = old_filename.replace(".md", "")

    # Remove common prefixes
    base = base.replace("execution-", "")
    base = base.replace("audit-complete-cde-mcp-", "audit-complete-")
    base = base.replace("EXECUTIVE_SUMMARY_AUDIT_", "audit-summary-")

    # Extract date
    date = extract_date_from_old_format(base)

    # Remove date from base
    base = re.sub(r"-?\d{4}-\d{2}-\d{2}", "", base)

    # Default time (12:00 if unknown)
    time = "1200"

    return f"EXECUTIONS-{base}-{date}-{time}.md"

# Run migration
execution_dir = Path("agent-docs/execution")
for file in execution_dir.glob("*.md"):
    if file.name.startswith("WEEK-"):
        continue  # Skip weekly summaries

    new_name = generate_new_filename(file.name)
    new_path = file.parent / new_name

    print(f"Renaming: {file.name} ‚Üí {new_name}")
    file.rename(new_path)
```

## üìä Benefits

### 1. **Consistency**
- Single naming pattern across all documentation
- Easy to remember and enforce

### 2. **Automation-Friendly**
- Predictable filenames enable automated processing
- Regex pattern matching for validation

### 3. **Chronological Sorting**
- ISO date format ensures correct sorting
- Time component allows multiple reports per day

### 4. **Clear Purpose**
- `EXECUTIONS-` prefix clearly identifies generated docs
- Distinguishes from permanent docs (specs, guides)

### 5. **Tooling Integration**
- Pre-commit hooks validate naming
- GitHub Actions process files by pattern
- MCP tools generate consistent filenames

## üö´ Exceptions

### Files NOT Following This Convention

**Permanent Configuration** (keep current names):
```
AGENTS.md           ‚Üê OpenAI standard (keep as-is)
GEMINI.md           ‚Üê Google AI Studio standard (keep as-is)
README.md           ‚Üê Project overview
CHANGELOG.md        ‚Üê Version history
CONTRIBUTING.md     ‚Üê Developer guidelines
```

**Reason**: Industry standards expect these specific filenames.

## ‚úÖ Validation Checklist

Before committing any generated documentation:

- [ ] Filename starts with `EXECUTIONS-`
- [ ] Title uses kebab-case (lowercase, hyphens)
- [ ] Date in ISO format (YYYY-MM-DD)
- [ ] Time in 24h format (HHmm)
- [ ] Extension is `.md`
- [ ] No spaces or special characters (except hyphens)
- [ ] Pattern matches: `^EXECUTIONS-[\w-]+-\d{4}-\d{2}-\d{2}-\d{4}\.md$`
